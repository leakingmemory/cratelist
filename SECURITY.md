# Security Policy

## Supported Versions

The following versions of `cratelist` are currently being supported with security updates:

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |
| < 0.1   | :x:                |

## Reporting a Vulnerability

We take the security of this project seriously. If you believe you have found a security vulnerability, please report it to us responsibly.

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please report them by sending an email to [sigsegv@radiotube.org](mailto:sigsegv@radiotube.org).

### What to include in your report

- A description of the vulnerability and its potential impact.
- Steps to reproduce the issue.
- Any relevant configuration details or environment information.
- (Optional) A proof-of-concept exploit.

### Our Response Process

1. We will acknowledge receipt of your report within 48 hours.
2. We will investigate the issue and confirm whether it is a security vulnerability.
3. If confirmed, we will work on a fix and determine a release schedule.
4. We will coordinate with you to disclose the vulnerability once a fix is available and has been tested.

## Security Update Policy

Security updates will be released as new patch versions (e.g., `0.1.1`) or as part of the next minor version (e.g., `0.2.0`). We recommend all users to update to the latest available version as soon as possible.
